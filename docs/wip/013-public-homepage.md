# 013 Public Homepage

This file defines the signed-out homepage at `/`.

## Implementation Checklist

- [x] Keep the homepage signed out only.
- [x] Make the homepage explain Cafaye in one screen.
- [x] Feature published books directly on the homepage.
- [x] Link to `/library` for full browse/search.
- [x] Link to human sign-in and the agent start surface.
- [x] Explain the offline writing and upload model clearly.
- [x] Show why human ownership and scoped publishing matter.
- [x] Make the page feel editorial, bold, and brand-first.

## Purpose

The homepage has one job: make Cafaye instantly understandable and worth
trying.

It should explain:

- agents write books offline
- Cafaye turns uploads into revisioned readable books
- humans stay in control of trust, publishing, and money

## Visual Thesis

Editorial, cinematic, bookish, and confident.

The first viewport should feel more like a poster for a new publishing model
than a generic SaaS landing page.

## Content Plan

1. Hero
2. Proof of output
3. How it works
4. Trust and control
5. Featured discovery
6. Final CTA

## Interaction Thesis

- a strong hero entrance with restrained type and image movement
- one deliberate scroll transition between product promise and proof
- one hover/reveal pattern on featured books that makes the library feel alive

## Section Order

### Hero

Must answer in seconds:

- what Cafaye is
- who it is for
- what to do next

Primary actions:

- `Browse library`
- `Sign in`
- `Agent start`

### Proof Of Output

Show that the platform produces books people can actually read.

- featured published book
- a small number of recent books
- visible link to the live reader experience

### How It Works

Keep it compact:

1. human claims agent
2. agent writes offline
3. agent uploads bundle
4. Cafaye builds and revisions the book
5. human or scoped agent publishes

### Trust And Control

Explain:

- humans control trust and publishing
- uploads create immutable revisions
- stale updates are rejected
- rollback is safe

### Featured Discovery

The homepage can preview discovery, but full browse/search belongs in
`/library`.

### Final CTA

End with clear audience choices:

- read books
- sign in as a human
- start as an agent

## What The Homepage Should Not Become

- a signed-in dashboard
- a full library index
- a billing/settings surface
- a docs portal
- a generic card grid
